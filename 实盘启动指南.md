# 实盘启动完整指南

## 🎯 第一步：准备工作（1-2周）

### 1.1 选择券商和开通权限

**推荐券商（支持程序化交易）：**
- 华泰证券（QMT）
- 银河证券（Ptrade）
- 广发证券（QMT）
- 国金证券（Ptrade）

**开通要求：**
- 账户资产50万以上（部分券商30万）
- 签署程序化交易协议
- 通过适当性评估
- 申请API接口权限

**时间：** 1-3个工作日

---

### 1.2 准备交易服务器

**方案A：本地电脑（不推荐）**
- 要求：24小时开机，网络稳定
- 风险：断电、断网、电脑故障

**方案B：云服务器（推荐）**
- 推荐：阿里云/腾讯云/华为云
- 配置：2核4G，100G硬盘，5M带宽
- 位置：选择离券商服务器近的节点（如上海、深圳）
- 成本：约100-200元/月
- 系统：Windows Server或Ubuntu

**部署清单：**
- [ ] 安装Python环境（3.9+）
- [ ] 安装依赖包（pandas, numpy, akshare等）
- [ ] 部署交易程序
- [ ] 设置定时任务（数据更新、策略运行）
- [ ] 配置监控告警（邮件/短信/钉钉）

## 🚀 第二步：模拟盘验证（2-4周）

### 2.1 模拟盘运行

**操作：**
- 使用券商模拟账户（通常有100万虚拟资金）
- 完全按策略信号执行
- 每天收盘后对比模拟成交与理论信号

**验证重点：**
- [ ] 信号生成是否及时
- [ ] 滑点是否在预期范围内（<0.1%）
- [ ] 成交价格是否合理
- [ ] 是否有信号丢失

**记录表格：**
```
日期 | 股票 | 信号时间 | 理论价格 | 模拟成交价格 | 滑点 | 备注
```

### 2.2 问题修复

**常见问题：**
- 数据延迟 → 更换数据源或服务器位置
- 滑点过大 → 避开流动性差的股票
- 信号丢失 → 检查程序逻辑和异常处理

## 🛡️ 第四步：风险控制体系

### 4.1 硬性风控规则

**自动执行：**
```python
# 1. 单日最大亏损限制
max_daily_loss = 0.03  # 单日亏损超过3%，停止当天交易

# 2. 总体回撤控制
max_total_drawdown = 0.20  # 总体回撤超过20%，减仓至50%
max_severe_drawdown = 0.25  # 回撤超过25%，停止新开仓

# 3. 单只股票亏损限制
max_single_loss = 0.10  # 单只股票亏损超过10%强制平仓

# 4. 仓位控制
max_single_position = 0.08  # 单只股票最大仓位8%
max_total_position = 0.70   # 总仓位不超过70%

# 5. 交易次数限制
max_daily_trades = 5  # 每天最多新开仓5只
```

### 4.2 人工干预条件

**需要人工判断是否干预：**
- 黑天鹅事件（如战争、金融危机）
- 策略连续亏损超过预期（如连续2周每天亏损）
- 技术故障（数据中断、API异常）
- 重大政策变化（如监管新规）

**不要干预：**
- 正常回撤（在预期范围内）
- 单日亏损（除非是极端行情）
- 个股正常波动

### 5.2 每周复盘

**复盘内容：**
1. 本周收益vs基准（沪深300）
2. 交易执行情况（是否有滑点异常）
3. 策略表现是否符合预期
4. 是否有需要优化的点

### 5.3 每月评估

**评估维度：**
- 收益是否达到回测预期的70%
- 最大回撤是否控制在30%以内
- 交易次数是否正常
- 是否有异常交易（如频繁止损某只股票）

**决策：**
- 如果连续3个月表现正常 → 加仓
- 如果连续2个月异常 → 减仓或暂停

## ⚠️ 第六步：应急预案

### 6.1 技术故障

**情况1：服务器宕机**
- 备用方案：本地电脑临时接管
- 恢复时间：<4小时

**情况2：数据中断**
- 备用数据源：切换至备用API
- 如果无法恢复：当天停止交易，次日恢复

**情况3：API异常**
- 联系券商技术支持
- 必要时转人工交易

### 6.2 市场极端情况

**情况1：大盘跌停（千股跌停）**
- 所有持仓无法卖出
- 处理：接受亏损，次日开盘视情况处理

**情况2：个股黑天鹅（如乐视、康美）**
- 连续跌停无法卖出
- 预防：分散投资，单股不超过8%

**情况3：策略失效**
- 连续3个月大幅跑输基准
- 处理：暂停策略，重新评估
