# 为什么设置了5%止损，还是有超过5%的亏损？

## 🔍 问题案例

**301018.SZ (申菱环境) 2025年4月7日**

| 时间 | 事件 | 价格 |
|------|------|------|
| 2025-03-31 15:00 | 买入 | 39.10 |
| 2025-04-03 15:00 | 前收盘价 | 37.52 |
| 2025-04-07 10:30 | 开盘价 | 33.84 (跳空-9.8%) |
| 2025-04-07 10:30 | 最低价 | 30.01 (比前收低20%) |
| 2025-04-07 10:30 | 收盘价 | 31.89 |
| **实际亏损** | **-18.44%** | - |

---

## 🎯 根本原因

### 回测逻辑的局限性

当前回测代码在每根K线**收盘后**才检查止损：

```python
for i in range(len(data)):
    row = data.iloc[i]
    close_price = float(row['Close'])  # 使用收盘价
    low_price = float(row['Low'])      # 使用最低价
    
    # 止损检查
    stop_price = pos.avg_cost * (1 - stop_loss_pct)  # 37.15 (5%止损)
    if close_price <= stop_price or low_price <= stop_price:
        should_sell = True
```

### 问题出在哪里？

1. **止损触发时机**：在10:30这根K线，最低价30.01已经远低于止损价37.15，应该立即止损

2. **但卖出价格**：代码使用`close_price = 31.89`作为卖出价

3. **实际亏损**：(31.89 - 39.10) / 39.10 = **-18.44%**

4. **预期亏损**：如果是实盘条件单，应该以开盘价33.84成交，亏损约-13.4%

---

## ⚠️ 这是回测精度问题，不是策略问题

### 实盘中会发生什么？

**情况1：条件单止损**
- 设置止损价37.15
- 4月7日开盘价33.84直接低于止损价
- 以开盘价33.84成交
- **实际亏损：约-13.4%**

**情况2：市价止损**
- 开盘即以市价卖出
- 可能以33.84或更低成交
- **实际亏损：-13% ~ -15%**

### 回测中发生了什么？

- 检测到低价30.01低于止损价37.15
- 但在该K线收盘后才执行卖出
- 卖出价为收盘价31.89
- **回测亏损：-18.44%**

---

## ✅ 如何修复？

### 方案1：跳空保护（推荐）

```python
# 在止损检查前添加跳空保护
if pos.shares > 0:
    stop_price = pos.avg_cost * (1 - stop_loss_pct)
    
    # 跳空低开保护：如果开盘价直接低于止损价，以开盘价卖出
    if open_price < stop_price:
        should_sell = True
        sell_reason = "stop_loss_gap"
        sell_price = open_price  # 以开盘价卖出
    # 正常止损检查
    elif close_price <= stop_price or low_price <= stop_price:
        should_sell = True
        sell_reason = "stop_loss"
        sell_price = close_price
```

### 方案2：使用更精确的止损逻辑

```python
# 如果最低价触及止损，假设以止损价成交（更乐观）
# 或者假设以(low + close)/2成交（折中）
if low_price <= stop_price:
    should_sell = True
    sell_price = max(stop_price, open_price * 0.99)  # 至少按止损价或开盘价99%成交
```

### 方案3：降低止损比例（保守）

```python
# 考虑到跳空风险，设置更严格的止损
stop_loss_pct = 0.03  # 3%止损，给跳空留一些空间
```

## 💡 建议

1. **接受这种误差**
   - 回测精度受限于K线周期（60分钟）
   - 分钟级数据无法捕捉盘中的精确成交
   - 这种跳空属于"黑天鹅"事件，任何策略都无法避免

2. **添加跳空保护**
   - 修改回测代码，在跳空低开时以开盘价卖出
   - 这样虽然仍有亏损，但更接近实盘

3. **降低仓位或分散投资**
   - 单只股票仓位不超过10%
   - 即使单笔亏损-18%，对整体组合影响有限

4. **添加大盘过滤器**
   - 当大盘（如沪深300）跳空低开超过2%时，暂停新开仓
   - 避免在系统性风险中频繁交易
